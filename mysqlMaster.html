学习mysql双主互备实现数据库同步

双主热备的概念简单说一下，就是要保持两个数据库的状态自动同步。对任何一个数据库的操作都自动应用到另外一个数据库，始终保持两个数据库数据一致。 这样做的好处多可以做灾备，其中一个坏了可以切换到另一个。还可以做负载均衡，可以将请求分摊到其中任何一台上，提高网站吞吐量。

一、准备环境

准备环境主服务器(Master)

IP 10.20.255.17
系统 Linux(SUSE LINUX 64位 v11)
MySQL版本 5.6.30
需要同步的数据库名 myslave
用于同步的帐号 slave001
用于同步的密码 123

准备环境从服务器(Master)

IP 10.20.255.151
系统 Linux(SUSE LINUX 64位 v11)
MySQL版本 5.6.30
需要同步的数据库名 myslave
用于同步的帐号 slave001
用于同步的密码 123

二、配置双Master服务器

两台服务器分别设置如下步骤

编辑my.cnf配置文件

vi /home/devp/software/mysql/conf/my.cnf
找到#server-id的位置，把以下内容对照设置进去，键入 i，激活输入状态
注意:这里的Master与Master的server-id不能相同

server_id=1
log_bin=/home/devp/software/mysql/data/mysql-bin.log
expire_logs_days=10
max_binlog_size=100M
binlog_do_db=myslave
binlog_ignore_db=mysql

auto_increment_increment=2
auto_increment_offset=1

log_slave_updates=1

replicate-ignore-db=mysql
replicate-do-db=myslave

server_id：Masterid是必须的，而且唯一。
Log_bin： 二进制日志。
expire_logs_days：过期最长天数。
max_binlog_size：日志文件最大限度
binlog_do_db：需要同步的数据库。如果需要同时同步多个数据库，请另起一行设置binlog_do_db:需要同步的数据库名称
binlog_ignore_db：需要忽略哪些数据库。
auto_increment_offset：表示自增长字段从那个数开始，他的取值范围是1 .. 65535
auto_increment_increment：表示自增长字段每次递增的量，其默认值是1，取值范围是1 .. 65535
log_slave_updates：表示slave将变成master。
replicate-ignore-db：复制时需要排除的数据库。
replicate-do-db：需要复制的数据库。

注意：编辑完成后按esc键输入:wq退出保存。

三、数据库同步

两台服务器分别设置如下步骤

1.建立数据库

mysql -u root -p
mysql> CREATE DATABASE myslave CHARSET ‘utf8′;

2.建立数据表

– —————————-
– Table structure for `users`
– —————————-
DROP TABLE IF EXISTS `users`;
CREATE TABLE `users` (
`id` mediumint(8) unsigned NOT NULL AUTO_INCREMENT,
`username` varchar(20) NOT NULL,
`website` varchar(30) DEFAULT NULL,
PRIMARY KEY (`id`)
) ENGINE=MyISAM DEFAULT CHARSET=utf8;

– —————————-
DROP TABLE IF EXISTS `users_log`;
CREATE TABLE `users_log` (
`user_log_id` mediumint(8) unsigned NOT NULL AUTO_INCREMENT,
`info` varchar(200) DEFAULT NULL,
`uid` mediumint(8) unsigned NOT NULL,
PRIMARY KEY (`user_log_id`)
) ENGINE=MyISAM DEFAULT CHARSET=utf8;

3.建立专门用户备份的用户

grant replication slave on .* to myslave001@’%’ identified by ‘123’;

4.重启mysql数据库

用root账号登录进入mysql查看master有没有生效

Show master status\G;如图所示。

File：就是二进制日志文件。
Position:表示当前的文件偏移量，就是写入在二进制日志文件的记录位置。

四、启动

两台服务器分别设置如下步骤

配置Slave信息并启动Slave服务

 CHANGE MASTER TO
MASTER_HOST=’10.20.255.17′,
MASTER_USER=’myslave001′,
 MASTER_PASSWORD=123′,
 MASTER_LOG_FILE=’mysql-bin.000001′,
MASTER_LOG_POS=106;

以上的是关键，需要填写访问ip,访问账号，访问密码和具体的MASTER_LOG_FILE以及MASTER_LOG_POS的数据，可以从Master服务器获得，如果不设置，将无法正常启动Slave服务

1.启动Slave服务

mysql> start slave;

2.查看同步状态

mysql> show slave status\G;如图所示。

当输入命令后会出现一大堆的状态，具体的看Slave_IO_Running以及Slave_SQL_Running的状态，如果都是Yes就代表正常
如果其中一个是No， 那就说明不成功。需要查看mysql的错误日志。

五、测试

在两台Master服务器分别插入条数据

mysql> use myslave;
mysql> INSERT INTO users(‘username’,’website’) VALUES(‘123’,’123’);

插入成功之后，就可以在另一台master服务器上查看到相同记录
如果插入不成功，放心，你可以再仔细检查一下本配置文档，是否有那一步有遗漏或没正确配置。
